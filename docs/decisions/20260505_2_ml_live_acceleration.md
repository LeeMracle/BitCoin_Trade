## 컨텍스트 — 가속 필요성

운영 1개월 누적 결과:
| 지표 | 값 |
|---|---|
| 누적 손익 | -55.66% |
| 승률 | 15.4% (2/13) |
| Profit Factor | 0.25 |
| Expectancy | -4.28%/거래 |

표준 흐름(3개월 Shadow + 1~2개월 A/B + LIVE)은 추가 5~6개월 적자 누적 위험. 사용자 "최대 효과 신속히" 요구 → **OOS 백테스트로 Shadow 데이터 부족 보완**.

## OOS 백테스트 결과 (2026-01-01 ~ 2026-03-31, 89일)

학습: 2022-01 ~ 2025-12-31 (3,258 샘플) / 검증: 학습 본 적 없는 2026-Q1
| 시나리오 | 거래 | 승률 | PF | Expectancy | 누적 |
|---|---|---|---|---|---|
| **무필터 (현재)** | 177 | 52.5% | 0.98 | -0.05% | **-8.6%** |
| score ≥ 0.40 | 127 | 50.4% | 0.98 | -0.06% | -7.6% |
| **score ≥ 0.45 (선택)** | **101** | **49.5%** | **1.04** | **+0.09%** | **+8.9%** |
| score ≥ 0.50 | 80 | 48.8% | 1.09 | +0.19% | +15.0% |
| score ≥ 0.55 (최적) | 62 | 48.4% | 1.17 | +0.34% | +21.1% |
| score ≥ 0.60 | 49 | 46.9% | 1.06 | +0.12% | +5.9% |

**0.45 선택 근거**: 0.55가 OOS 최적이지만 거래량 35% (1주 5건)로 큰 변동. 점진적 도입 위해 **0.45 시작 (57% 통과, 1주 ~8건)** → 1주 안정 후 강화.

## 결정

### 변경
1. AWS systemd drop-in (`/etc/systemd/system/btc-trader.service.d/ml.conf`):
   ```
   Environment="ML_FILTER_ENABLED=1"
   Environment="ML_SHADOW_MODE=0"          ← 1 → 0 (차단 활성)
   Environment="ML_FILTER_THRESHOLD=0.45"  ← 신규
   ```
2. 봇 재시작 → score < 0.45 매수 신호 차단

### 예상 효과 (OOS 추정)
- 매수 빈도: ~57% (177 → 101 시뮬 비율)
- PF: 0.98 → 1.04 (적자 → break-even 이상)
- Expectancy: -0.05% → +0.09%/거래

### 주간 점진 강화 일정 (조건부)
| 일자 | threshold | 조건 |
|---|---|---|
| 5-5 | **0.45** | 즉시 시작 |
| 5-12 | 0.50 | 1주 실거래 PF ≥ 1.0 + 매수 빈도 ≥ 3건/주 |
| 5-19 | 0.55 | 추가 1주 PF ≥ 1.1 |
| 6월 | 시드 증액 검토 | PF ≥ 1.2 안정 시 |

## 위험 평가

| 위험 | 완화 |
|---|---|
| OOS 단일 환경 (Q1 2026 횡보장) → 다른 환경 효과 미확인 | 0.45 보수 시작 + 매주 평가 |
| 백테스트 환상 (슬리피지 0.2% 가정만) | 실거래 1주 모니터링 후 강화/롤백 결정 |
| 매수 빈도 급감 → 표본 부족 | 0.45는 57% 통과로 적정 (0.55는 35%) |
| 모델 drift 가능성 | outcome 매칭 cron 기존 동작 (P8-21) |
| 잘못된 차단으로 큰 기회 상실 | 매일 차단 신호 outcome 추적 |

## 롤백 계획

문제 발생 시 **1줄 변경으로 즉시 복귀** 가능:
```bash
ssh ... 'sudo sed -i s/ML_SHADOW_MODE=0/ML_SHADOW_MODE=1/ /etc/systemd/system/btc-trader.service.d/ml.conf && \
         sudo systemctl daemon-reload && sudo systemctl restart btc-trader'
```

조건:
- 1주 누적 -10%+ 손실 → 즉시 SHADOW 복귀
- 1주 매수 0건 → threshold 너무 높음 (0.40으로 완화)
- ML 추론 latency 1초+ 빈발 → fail-open 검토

## 성공기준 (1주 후 5-12 평가)

- [ ] 1주 매수 시도 ≥ 3건 (현재 0건/30h)
- [ ] 1주 PF ≥ 1.0 (실거래)
- [ ] ML 차단 비율 30~50% (95% 미만, 5% 이상)
- [ ] 차단 신호 outcome (도달 vs 미도달) — TN 비율 ≥ 50%
