## 1. 결정 배경

### 1.1 현재 성과 (closed 5건 누적)
| 지표 | 실측 | 백테스트 목표 |
|---|---|---|
| 승률 | **20%** (1/5) | 35%+ |
| 평균 손실 | **-14.2%** | <-7% |
| 평균 수익 | +18.1% | >+0.5% |
| 누적 P/L | **-38.8%** | +양수 |

### 1.2 핵심 문제 진단
1. **추세추종 본질 — 승률 낮음 + 큰 수익 1~2건이 손실 다수 커버**해야 함
2. **현재는 큰 수익 종목조차 트레일 스탑 한 번에 전량 매도 → 평균 수익 변동 ↑**
3. **DC 돌파 신호의 가짜 돌파 비율 높음** — 거래량 동반 확인 없음
4. **단일 사건(ONG -29.2%) 후속 손실 방어 메커니즘 없음** — 5연패 자동 중단은 너무 늦음

### 1.3 DC/ATR 단독 조절의 한계
- DC 기간(15→10→15) 변경은 **신호 빈도** 조절일 뿐 **신호 품질**은 동일
- ATR 배수 변경은 **손절 거리** 조절일 뿐 **승률**과 무관
- → 진입·청산·위험 관리 **3축 모두에 새 변수 추가 필요**

## 2. 채택 사항

### 2.1 부분 익절 (Tiered Take-Profit)
**기존**: 트레일 스탑 깨질 때까지 보유 → 전량 매도
**변경**: 단계별 분할 익절
- +5% 도달 → 30% 매도
- +10% 도달 → 30% 매도
- +15% 도달 → 30% 매도
- 잔량 10% → 트레일 스탑 (기존 ATR×3 + 하드캡)

**기대 효과**:
- ONT +18.1% 케이스: 기존 = 18.1% 한 번. 신규 = 5+10+15+18 / 4 = **+12% 평균이지만 변동성 ↓**
- 큰 수익 종목의 **익절 타이밍 안정화**
- 자비스 BTC SELL 패턴(이미 검증됨) composite로 확장

### 2.2 거래량 필터 (Volume Confirmation)
**기존**: DC15 돌파만으로 매수
**변경**: DC15 돌파 + **거래량 5일 평균 × 1.5배 이상**
- daytrading 모드는 이미 적용 중 (`_DT_VOL_THRESHOLD`)
- composite 모드에도 동일 적용

**기대 효과**:
- 가짜 돌파(소량 거래) 50%+ 차단
- 진입 빈도 약 30~40% 감소 (대신 품질 ↑)
- 승률 35% → 45~50% 가능 (이론, 백테스트 검증 필요)

### 2.3 일일 손실 한도 (Daily Loss Circuit)
**기존**: 5연패 자동 중단 (봇 종료 — 너무 가혹)
**변경**: 24h 누적 실현손익 -3% 도달 시 신규 매수 24h 중단
- 자동 해제: 매일 09:00 KST에 daily_pl_state 리셋
- 중단 중에도 기존 포지션 트레일링 스탑은 정상 작동
- 5연패 자동 중단은 **유지** (별도 안전장치)

**기대 효과**:
- ONG -29.2% 같은 단일 큰 손실 후 추가 매수 차단 → 누적 -38% → -25% 정도로 완화 가능
- 일중 변동성 큰 날 방어 (BTC -5%+ 같은 macro 충격)
