# ADR: BATA v2.0 — AI 자율 판단 매매 시스템

> 작성: 2026-04-06 | 상태: **브레인스토밍** | KATA v2 방향 동기화

## 배경

현재 BATA v1.5는 규칙 기반 전략 + config.py 수동 변경 방식이다.
- 12개 전략 레지스트리 보유 (composite, dc_atr, regime, ensemble 등)
- 전략/파라미터 변경 시 사람이 config.py를 직접 수정해야 함
- 시장 이슈(관세, 규제, 금리) 발생 시 자동 대응 불가능
- 24/7 시장이므로 야간 급변 시 대응 공백 발생

KATA(주식 자동매매)가 v2.0(AI 자율 판단)으로 전환 결정했으며,
BATA도 동일한 방향으로 업그레이드한다.

## 결정

**BATA를 v2.0(AI 자율 판단 매매)으로 업그레이드한다.**

- Claude API(Sonnet)를 판단 엔진으로 사용
- 정기 판단 2회/일 + 변동성 자동 트리거 + 사용자 긴급 트리거
- 리스크 안전장치 완비 후 v2.5(완전 자동) 전환

## 버전 로드맵

```
v1.0 (과거)   Composite DC20 단일 전략
    ↓
v1.5 (현재)   9년 데이터 재검증 + 전략 다각화 + 서킷브레이커
    ↓
v2.0 (이번)   Claude가 시장 분석 → 전략/파라미터 추천 → 텔레그램 승인 후 적용
    ↓
v2.5          Claude 추천 → 자동 적용 (리스크 장치 완비 조건)
    ↓
v3.0          완전 자율 (이상 감지 시만 사람 개입)
```

## KATA vs BATA 핵심 차이

| 구분 | KATA (주식) | BATA (비트코인) |
|------|------------|----------------|
| 시장 | 한국 주식, 장중 6시간 | **24/7 무중단** |
| 거래소 | 키움증권 REST API | 업비트 (현물, KRW) |
| 종목 수 | 코스피/코스닥 2000+ | KRW 마켓 ~243개 |
| 방향 | 롱/숏 가능 | **롱/플랫만** (숏 불가) |
| 현재 수준 | 규칙 기반 레짐 + 고정 전략 | **12개 전략 레지스트리** + composite |
| 뉴스 영향 | 종목별 공시/실적 | **매크로 중심** (금리, 관세, 전쟁, 규제) |
| 판단 타이밍 | 08:30, 13:00 (장중) | 09:00, 21:00 + **변동성 자동 트리거** |

## 판단 주기

| 구분 | 시간 | 내용 |
|------|------|------|
| 모닝 판단 | 09:00 KST | 전일 글로벌 이슈 + 오늘 전략 결정 |
| 이브닝 판단 | 21:00 KST | 미국장 개장 전후 + 야간 전략 조정 |
| 긴급 판단 | 수시 | 사용자 `/signal` 트리거 → 즉시 판단 |
| 자동 감시 | 상시 | BTC ±5% 변동 시 자동 트리거 |

## Claude 판단 엔진 입출력

### 입력 (프롬프트)

```
- BTC/KRW 일봉 최근 30일 + 4시간봉 최근 7일
- 현재 포지션 상태 (보유 종목, 평단가, 수익률)
- F&G 지수 + BTC 도미넌스 + 김치 프리미엄
- 펀딩레이트 (바이낸스 참조)
- 뉴스 요약 (핵심 5줄 이내)
- 현재 적용 전략 + 최근 7일 성과
- 전략 카탈로그 (선택 가능한 전략 목록 + 특성)
```

### 출력 (JSON)

```json
{
  "regime": "bull|neutral|bear|crisis",
  "strategy": "composite|dc_atr|regime|vol_reversal|...",
  "strategy_kwargs": {"dc_period": 20},
  "max_positions": 5,
  "position_ratio": 0.95,
  "vb_enabled": true,
  "reasoning": "미국 관세 발표로 단기 하락 예상, 방어적 전략 전환",
  "confidence": 0.75,
  "next_review": "2026-04-06T21:00:00+09:00"
}
```

| 장치 | 설명 |
|------|------|

| v1 폴백 | Claude API 장애 시 기존 규칙 기반 유지 |

## v2.0 → v2.5 전환 기준

기간이 아닌 **신뢰도 + 안전장치 완성도** 기준:
- 안전장치 전체(손실한도, 서킷브레이커, 긴급정지, 쿨다운) 구현 완료
- v2.0 승인 모드에서 판단 정확도 체감 확인
- 판단 로그 최소 30회 이상 축적 후 사후 평가
- 사용자 판단으로 전환 (별도 기간 제한 없음)

## BATA의 강점 (즉시 활용 가능)

1. **12개 전략 레지스트리** — Claude가 전략을 "선택"만 하면 됨 (신규 개발 불필요)
2. **config.py 한 줄 변경**으로 전략 전환 가능한 구조
3. **composite 전략**이 이미 레짐 스위칭 내장 → AI 판단과 자연스럽게 결합
4. **텔레그램 알림 인프라** 기존 구축 완료 → 승인 UI 확장만 필요
5. **AWS 배포 체계** 이미 구축 → 크론 추가만 필요
