# 모니터링 보고 평가금액에 알트코인 누락

- **발생일**: 2026-04-05
- **심각도**: MEDIUM
- **카테고리**: 모니터링 / 데이터 정합성

## 증상

텔레그램 모니터링 보고에서 "평가금액 187,933원"으로 표시되었으나, 업비트 앱에서 확인한 실제 총 평가액은 약 282,884원. 사용자가 불일치를 발견.

## 원인

`get_balance()` 함수(`services/execution/upbit_client.py`)가 **KRW + BTC만 반환**하고, 봇이 보유 중인 알트코인(TRX, ONT, RVN, CHZ, ONG 등)의 평가액을 합산하지 않음.

모니터링 보고가 이 함수의 `total_krw` 값을 그대로 사용하여 실제보다 ~94,000원 과소 보고.

추가로 BTC는 수동 매수분으로 봇 상태파일에 미등록 상태였음.

## 수정

- [x] `get_balance()` 또는 모니터링 보고 로직에서 전체 보유 자산 평가액 합산 — hourly_monitor.py 수정 완료 (04-05)
- [x] 봇 미추적 종목(수동 매수분) 표시 로직 추가 — 전체 자산 합산 + 현재가 기준 수익률 표시 (04-05)

## 검증 규칙

- 모니터링 보고 평가금액 = 업비트 API `fetch_balance()` 전체 자산 합계와 일치해야 함
- 봇 추적 종목 ≠ 업비트 실제 보유 종목이면 불일치 경고 표시

## 교훈

모니터링 보고의 핵심 수치(평가금액)는 반드시 거래소 API 전체 자산 조회 기반이어야 한다. 부분 합산은 사용자 신뢰를 해치고 의사결정을 왜곡한다.
