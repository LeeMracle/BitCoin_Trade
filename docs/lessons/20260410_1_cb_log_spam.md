# L14: 서킷브레이커 로그 스팸 (6,420건/일)

## 날짜
2026-04-10

## 증상
- `btc-trader.service` journalctl 로그에 `[서킷브레이커] 발동 중 — MON/KRW 매수 차단` 메시지가 초당 수회 반복
- 04-10 00:00~05:20 사이 **6,420건** 누적
- 중요 로그(VB, EMA, 매매 신호)가 CB 스팸에 묻힘

## 원인
- `_execute_buy()`가 웹소켓 체결 이벤트마다 호출됨
- CB 발동 상태에서 `is_triggered()` / `is_l2_triggered()` 체크 후 **매번** print 실행
- 146개 종목 × 초당 수회 체결 = 로그 폭발

## 수정
- `realtime_monitor.py`에 `_cb_log_ts` (monotonic 타임스탬프) 필드 추가
- CB "발동 중" 로그를 **60초 throttle**로 제한 (60초 내 중복 출력 억제)
- 수정 후 2분간 CB 로그 2건 (정상)

## 검증규칙
- `realtime_monitor.py`에서 반복 호출 경로(웹소켓 콜백 → `_execute_buy`)에 있는 print문은 throttle 필수
- pre_deploy_check: `_cb_log_ts` 필드 존재 확인

## 교훈
- 실시간 이벤트 루프 안의 로그는 **항상 throttle/dedupe 적용** — 종목 수 × 이벤트 빈도가 곱해지므로 1건처럼 보이는 로그도 수천 건이 될 수 있음
- 배포 후 journalctl 로그 볼륨을 반드시 확인할 것
