# 실시간 틱 진입 vs 봉 마감 진입 괴리

- **발생일**: 2026-03-29
- **심각도**: HIGH
- **카테고리**: 전략

## 증상

daytrading(DC15+Vol2.5x+Trail2%) 실전 투입 결과: 8건 중 1승 7패 (12.5%), 6연패.
위꼬리(가짜 돌파)에 진입 → 되돌림에 스탑 히트 반복.

## 원인

- **백테스트**: 봉 마감 종가 기준 → 다음 봉 시가 체결 (look-ahead bias 방지)
- **실전**: 웹소켓 틱이 DC 상단 돌파 즉시 체결 → 봉 내 가짜 돌파에 잡힘
- 4시간봉 전략을 실시간 틱으로 실행하면 전혀 다른 전략이 됨

## 수정

`services/execution/realtime_monitor.py` — 웹소켓 틱 감시 시 봉 마감 확인 로직 추가:
- 일봉 전략: 09:00 KST 캔들 확정 후 진입 판단
- 4시간봉 전략: 4시간 주기 스캔 모드 사용 (실시간 틱 금지)

## 검증규칙

- 전략의 타임프레임과 실행 모드가 일치하는지 확인
- 일봉 전략 → `realtime_monitor.py`에서 캔들 확정 대기 로직 존재 확인
- 4시간봉 전략 → 스캔 모드(`--scan-interval`) 사용 확인

## 교훈

백테스트 타임프레임과 실전 실행 타임프레임은 반드시 일치해야 한다.
봉 마감 기반 전략을 실시간 틱으로 실행하면 가짜 돌파에 반복 피해를 입는다.
