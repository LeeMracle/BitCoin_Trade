2. 레이어별 책임

📦 Data Layer 기존

  • BT 결과: data/cache.duckdb 백테스트 trade 테이블
  • 실시간 OHLCV: ccxt 1m/15m/1d
  • 실거래 로그: workspace/jarvis_*.json

🔧 Feature Store 신규

  • 위치: data/features/{date}.parquet
  • 30~40 features, 학습/추론 동일 함수
  • Lookahead 방지: 봉 마감 후 계산 강제

🧠 Training 신규

  • Walk-forward CV: 분기별 fold 6~8개
  • XGBoost / LightGBM binary classifier
  • 슬리피지 0.2% 가정으로 라벨 보정

⚡ Inference 신규

  • multi_trader 매수 직전 hook
  • p95 latency < 50ms (모델 메모리 상주)
  • Threshold 0.55 (튜닝 가능)

💰 Execution 수정

  • multi_trader.py 매수 분기에 ML gate 삽입
  • 점수 × 기본수량 동적 사이징 (옵션)
  • 차단 사유 로깅 필수 (shadow 비교용)

📊 MLOps 신규

  • Shadow mode 3개월 → A/B → 채택
  • PSI/KS drift 주간 리포트
  • 월 1회 재학습 (cron)

10. 예시 코드 스켈레톤

services/ml/inference.py

import joblib, json
from pathlib import Path
from services.ml.features import compute_features

class MLFilter:
    def __init__(self, model_path: Path, threshold: float = 0.55):
        self.model = joblib.load(model_path)
        self.meta = json.loads((model_path.with_suffix('.meta.json')).read_text())
        self.threshold = threshold

    def score(self, symbol: str, ohlcv_df, market_ctx: dict) -> float:
        x = compute_features(symbol, ohlcv_df, market_ctx)  # 학습과 동일 함수
        return float(self.model.predict_proba([x])[0][1])

    def passes(self, score: float) -> bool:
        return score >= self.threshold

services/execution/multi_trader.py (수정 패치)

# 매수 분기 직전
score = ml_filter.score(symbol, ohlcv_df, market_ctx)
shadow.log(symbol, score, signal_type="DC15", will_buy=ml_filter.passes(score))

if not ml_filter.passes(score):
    notifier.send(f"🚫 ML 차단 {symbol} score={score:.2f}", level="info")
    return

# 기존 매수 로직 그대로
position_size = base_size * (0.7 + 0.6 * score)  # 0.55→1.03×, 1.0→1.3×
upbit_client.buy(symbol, position_size)
