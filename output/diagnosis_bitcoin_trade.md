## 5. 강점 / 약점 요약

### 컨텍스트 (C)
- **강점**: 보안 경계와 금지사항이 CLAUDE.md + .gitignore에 이중으로 명문화. 교훈(lessons)이 인덱스·본문 양쪽에서 시스템화.
- **약점**: 사전 실행계획 저장소 부재(C7) — 사후 결정 문서만 있고 작업 착수 시점 계획이 휘발됨.

### 도구·환경 (T)
- **강점**: `pre_deploy_check.py`로 배포 전 자동 검증 + lessons와 연동된 룰 누적 구조. 시크릿/권한 최소화 모범적.
- **약점**: 훅(SessionStart/PreToolUse) 미사용으로 "문서로만 강제"되는 항목이 다수. 사람이 잊으면 무력화.

### 평가 (E)
- **강점**: 트레이딩 산출물(백테스트, 일일보고, 의사결정)의 사후 기록·회고 루틴이 견고. 멀티에이전트 분업 구조 완비.
- **약점**: **자기 세션 자기평가 편향 차단 메커니즘 부재**(E4) — 가장 큰 품질 리스크. 에이전트 운영 메트릭 미수집(E6).

## 6. 개선방향 Top 3

영향도(품질·재발 방지·자동화) × 작업 난이도를 고려한 우선순위.

### ① Execution Plan 강제 저장 체계 [C7 / E1·E3 동시 해소]

- **(a) 현재**: 작업 착수 시 계획이 휘발. 사후 `docs/decisions/`에 결과만 남음. 회고 시 "계획이 틀렸는지, 실행이 틀렸는지" 분리 불가.
- **(b) 목표**: 모든 비자명 작업은 시작 시점에 계획 + 성공기준을 파일로 남긴다.
- **(c) 산출물**:
  - `workspace/plans/YYYYMMDD_작업명.md` 디렉토리 신설
  - 템플릿: 목표 / 성공기준(체크박스 N개) / 단계 / 리스크 / 종료 후 회고 1줄
  - CLAUDE.md "## 작업 규칙"에 "비자명 작업은 plans/에 사전 계획 파일 생성 필수" 추가
  - (선택) `.claude/settings.local.json` SessionStart 훅으로 안내문 출력
- **(d) 작업량**: **소** (디렉토리 + 템플릿 1개 + CLAUDE.md 3줄 추가)

### ② 교차검증 운영 규칙 도입 [E4 — 가장 큰 품질 리스크]

- **(a) 현재**: 같은 세션이 자기 작업을 평가. 자기평가 편향 미차단. `cto` 스킬은 존재하나 교차모델/세션 강제 아님.
- **(b) 목표**: 코드/전략 변경 시 "구현 세션 ≠ 검토 세션"을 규칙화. 가능하면 다른 모델 또는 별도 서브에이전트가 검토.
- **(c) 산출물**:
  - CLAUDE.md "## 작업 규칙"에 항목 추가:
    > "구현을 수행한 동일 세션이 자기 산출물을 PASS 판정하지 않는다. 검토는 (1) 별도 세션 (2) 별도 서브에이전트(예: cto review) (3) 자동 검증 스크립트 중 최소 1개로 수행한다."
  - `cto` 스킬의 `review`/`gate` 단계를 배포 전 필수 단계로 격상 (이미 존재하는 스킬 활용 → 신규 개발 최소)
  - 회고 항목에 "이번 작업 검증 주체" 1줄 기록
- **(d) 작업량**: **소** (규칙 명문화 + 기존 스킬 활용)

### ③ 운영 메트릭 측정 + Hook 자동화 [E6·E10 / T4·T9]

- **(a) 현재**: 트레이딩 성과는 측정하나, 에이전트 운영(세션 비용·시간·승인횟수·실패 시점·하네스 변경)은 미측정. 강제 사항이 문서에만 존재.
- **(b) 목표**: 최소한의 운영 메트릭을 자동 수집하고, 핵심 강제 사항은 훅으로 차단.
- **(c) 산출물**:
  - `workspace/agent_metrics.jsonl` — 일일작업 종료 시 (날짜, 작업명, 소요, 검증주체, 실패유무, 교훈ID) 1줄 append
  - `daily-work end` 스킬에 위 append 단계 추가
  - `.claude/settings.local.json`에 PreToolUse 훅: 배포·삭제 명령 패턴 매칭 시 `pre_deploy_check.py` 실행 강제
  - SessionStart 훅: `workspace/plans/`의 미완 계획 목록 출력
- **(d) 작업량**: **중** (훅 작성·테스트 + 메트릭 스키마 합의)
